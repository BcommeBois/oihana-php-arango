## [Unreleased]

### Added

- Negated string filter operators `nsw` / `new` / `ncontains` / `nregex` ‚Äî the `n`-prefixed negations of `sw` / `ew` / `contains` / `regex`, wrapping the positive function form in `!( ‚Ä¶ )` via the existing `logicalNot()` operator helper (e.g. `nsw` ‚Üí `!(STARTS_WITH(doc.x, @value))`). New `FilterComparator::NSW`/`NEW`/`NCONTAINS`/`NREGEX`. Documented in `wiki/{fr,en}/db/filter.md`.

- String filter operators `contains` and `regex` ‚Äî `CONTAINS(doc.x, @value)` (literal substring) and `REGEX_TEST(doc.x, @value)` (ICU regular expression). Both bound (no AQL injection) and added as function-form arms of `prepareFilterString`. `contains` is case-insensitive via the `alt` mirror; `regex` via an inline `(?i)` flag in the pattern (a client-supplied regex can be ReDoS-prone ‚Äî validate untrusted input on the application side). New `FilterComparator::CONTAINS` / `FilterComparator::REGEX`. Documented in `wiki/{fr,en}/db/filter.md`.

- String filter operator `ew` (ends with) ‚Äî `?filter={"key":"name","op":"ew","val":"leon"}` ‚Üí `RIGHT(doc.name, CHAR_LENGTH(@value)) == @value`. AQL has no native `ENDS_WITH`, so the suffix is matched literally (no `LIKE` pattern, nothing to escape), symmetric to `sw`. Case-insensitive via the `alt` mirror (`RIGHT(LOWER(doc.name), ‚Ä¶) == LOWER(@value)`). New `FilterComparator::EW`; `prepareFilterString` now dispatches its function/range forms (`between`/`sw`/`ew`) through a `match`. Documented in `wiki/{fr,en}/db/filter.md`.

- String filter operator `sw` (starts with) ‚Äî `?filter={"key":"name","op":"sw","val":"eka"}` ‚Üí `STARTS_WITH(doc.name, @value)`. Function-form operator (like `between`), not an infix comparator: the prefix is matched literally (no `%`/`_` wildcards, nothing to escape) and can use a persistent index. Case-insensitive via the `alt` mirror (`STARTS_WITH(LOWER(doc.name), LOWER(@value))`). New `FilterComparator::SW`. Documented in `wiki/{fr,en}/db/filter.md`.

- Array filter quantifiers unified under the `quant` key (`?filter=`) ‚Äî answers the element axis ¬´ how many elements must match ¬ª independently of the comparator (which stays in `op`/`match`). Values: `any` (default) / `all` / `none` / an integer `n` (= at least `n`). One vocabulary drives both array surfaces: scalar arrays via the AQL array comparison operator (`doc.scores ALL >= @v`, `doc.scores AT LEAST (2) >= @v`) and object arrays via the AQL question-mark operator (`doc.reviews[? AT LEAST (3) FILTER CURRENT.rating >= @v]`, also with `match`), the latter making `ALL`/`NONE`/`AT LEAST` expressible where only the existential `LENGTH(...) > 0` was possible before. New `FilterParam::QUANT`, `FilterQuantifier` enum, and free helper `resolveQuantifier()` (threshold cast to int, injection-safe; unknown quantifier ‚Üí `ValidationException`). Backward compatible: omitting `quant` keeps the legacy AQL unchanged, and the legacy scalar notations `op:"all.ge"` / `op:["atLeast.ge", n]` remain valid aliases. `quant` applies to first-level object arrays only (nested `a[*].b[*]` keeps the existential behaviour). Documented in `wiki/{fr,en}/db/filter.md`.

- HTTP facets (`?facets=`) overhaul ‚Äî every facetable field is declared under `Arango::FACETS` and reuses the filter operator vocabulary (`FilterComparator` / `FilterArrayComparator` + `FilterParam::OP/VAL`), with new and generalised types:
  - `Facet::IN` ‚Äî array-membership facet (op-driven, default `any.in`); the former `LIST`, `LIST_FIELD`, `LIST_FIELD_SORTED` are kept as aliases.
  - `Facet::JOIN` ‚Äî simple key-join existential facet (the join counterpart of `Facet::EDGE`): keeps documents having at least one joined document whose field matches the requested value, with multi-value OR and `-` negation.
  - `Facet::JOIN_COMPLEX` ‚Äî key-join existential facet matching SEVERAL fields (AND) on the same joined document (object `{field: condition}`), sharing the new `HasFacetComplexConditions` builder with `EDGE_COMPLEX`.
  - `Facet::EDGE` generalised ‚Äî configurable operator (`Facet::OP`) and multi-field OR (`AQL::FIELDS`), multi-value CSV (OR) and `-` negation as exclusion (`LENGTH(‚Ä¶) == 0`). The multi-field `like` configuration replaces the former THESAURUS facet.
  - The URL-provided sub-field names of complex facets (interpolated into `doc.<field>`) are validated by the new `isAttributeName` / `assertAttributeName` helpers ‚Äî a dangerous name fails the facet (logged and skipped), so no untrusted fragment ever reaches the AQL.
- `alt` symmetric transformations on both `?filter=` and `?facets=` ‚Äî `alt` now wraps the comparison **value** (right side), not only the **field** (left). The object form `alt:{ key:<chain>, val:<chain> }` gives one chain per side, and `val:true` mirrors the key-side chain (e.g. `LOWER(doc.email) == LOWER(@v)`); an array value maps the chain over each element (`@v[* RETURN LOWER(CURRENT)]`). On `?filter=` it covers the flat filters (string/number/array/date) and the nested array-expansion filters (`field[*].sub` and `match` ‚Äî `LOWER(CURRENT.email) == LOWER(@v)`, with `alt` applied globally to every `match` sub-field); edge/join/document leaves inherit it through the underlying flat filter, while the structural join key stays raw. On facets the chain may come from the model definition (new `Facet::ALT`, a default for every request) or from the URL request object (`{op,val,alt}`, which takes precedence). It is wired on `FIELD`, `EDGE`, `JOIN` and `IN` (membership ‚Äî where the array field is projected element-wise, `doc.tags[* RETURN LOWER(CURRENT)]`); complex facets (`EDGE_COMPLEX` / `JOIN_COMPLEX` / `ARRAY_COMPLEX`) accept a global `Facet::ALT` applied to every sub-field. The legacy field-only string/list forms (`"lower"`, `["trim","lower"]`) are unchanged. Backed by the shared free helpers `oihana\arango\db\helpers\alterExpression()` / `resolveAltSides()`, called directly by the filter and facet builders; documented with per-case examples in `wiki/{fr,en}/db/{filter,facets}.md`.
- New `AT LEAST (n)` array quantifier for `?filter=`: declared with the array-form operator `["atLeast.<cmp>", n]` (element 0 the `atLeast.<cmp>` code, element 1 the threshold, defaulting to 1), it keeps documents where at least `n` array elements satisfy the comparison ‚Äî e.g. `{"op":["atLeast.ge",2],"val":80}` ‚Üí `doc.scores AT LEAST (2) >= @v`. The `<cmp>` suffix reuses the filter operators (eq/ne/gt/ge/lt/le/in/nin); the threshold is cast to an int (injection-safe) and the field stays `alt`-aware. New `FilterArrayComparator::AT_LEAST*` constants.
- New `between` (range) operator for `?filter=` (number/string/date) and the `?facets=` FIELD facet: an inclusive `key >= @min && key <= @max` driven by `min`/`max` keys. Bound omission is type-driven ‚Äî number/string drop the omitted side (one-sided range), while a date resolves an omitted bound to "now" (always two-sided); a date `tz` applies to both bounds (`DATE_LOCALTOUTC(@min,@tz)`). The compared field stays `alt`-aware. Backed by the shared, unit-tested `oihana\arango\db\helpers\buildBetweenClauses()` helper.
- New `Field::ALTERS` output-side transformation on the field projection: an `alt`-style chain (same `FilterFunction` vocabulary ‚Äî `'lower'`, `['trim','lower']`, `['substring',0,3]`) wrapping the projected value at `RETURN` time, e.g. `'name' => [ Field::ALTERS => ['trim','lower'] ]` ‚Üí `name: LOWER(TRIM(doc.name))`. The filter-side `alt` counterpart for the output, it homogenizes filters ‚Üî projection. Opt-in per field, applied only to the default scalar projection (`key: doc.key`) ‚Äî ignored on typed/structural filters (BOOL, DATETIME, EDGE, JOIN, MAP, ‚Ä¶); `Field::NAME` still selects the source attribute so a transformed value can be exposed under another key. Documented with worked examples in `wiki/{fr,en}/edges-joins-projection.md`.
- New aggregate facets `Facet::EDGE_AGGREGATE` and `Facet::JOIN_AGGREGATE` for `?facets=`: instead of the existential `LENGTH(‚Ä¶) > 0` of `EDGE`/`JOIN`, they aggregate a numeric field over all linked documents and compare it to a threshold ‚Äî `AGG(FOR doc_x IN <source> [FILTER <join>] RETURN doc_x.<field>) <op> @threshold`. Driven by the request object `{agg, field, op, val}` (a bare scalar = threshold), each piece overridable on the URL and falling back to the definition (`Facet::AGG`, `AQL::FIELDS`, `Facet::OP`). The aggregator (`avg`, `sum`, `min`, `max`, `count`) is whitelisted by the new `FacetAggregator` enum (unknown ‚áí facet skipped + logged); `count` is field-less (`RETURN 1`) and generalizes the existential. A URL-provided `field` is validated with `assertAttributeName`, the threshold is numeric-cast, and the comparison is guarded by `LENGTH(‚Ä¶) > 0` so a `lt`/`le` threshold never lets the AQL empty-set sentinel (`null` for avg/min/max, `0` for sum/count) match documents with no linked document. New keys `Facet::AGG`, `FilterParam::AGG`, `FilterParam::FIELD`; shared `HasFacetAggregateConditions` builder; documented with worked examples in `wiki/{fr,en}/db/facets.md`.
- New conditional `alt` functions `coalesce` / `notNull`: wrap the field in AQL `NOT_NULL(...)` to substitute a default when it is missing or null before comparison ‚Äî e.g. `alt:[["coalesce",0]]` ‚Üí `NOT_NULL(doc.discount, 0) == @v`. The default value(s) come from the URL and are inlined as strict AQL literals via `json_encode` (always quoted/escaped, never the raw passthrough of `aqlValue()`), so a default is always treated as literal data ‚Äî injection-safe.
- New `pluck` `alt` function (and the `oihana\arango\db\functions\arrays\pluck()` helper): projects an array of objects onto a single sub-field via AQL's inline projection `array[* RETURN CURRENT.<field>]`. Chained with an aggregate it reduces over a property of embedded objects ‚Äî e.g. `alt:[["pluck","price"],"avg"]` ‚Üí `AVERAGE(doc.items[* RETURN CURRENT.price])`. The sub-field name comes from the URL and is validated with `assertAttributeName` before interpolation (injection-safe; an unsafe name throws `ValidationException`).
- New AQL array-expansion helpers `oihana\arango\db\functions\arrays\arrayMap()` (`array[* RETURN expr]`) and `arrayFilter()` (`array[* FILTER cond]`), centralizing the previously hand-built `[* ‚Ä¶ ]` string literals. The five existing sites now build through them ‚Äî `pluck()` (a thin `arrayMap` wrapper over a validated sub-field), the array-valued `alt` projection in `FilterTrait`, the `Facet::IN` field-side projection in `HasFacetIn`, and the two array-expansion existentials in `HasFilterArray` (`LENGTH(arrayFilter(...)) > 0`). Pure refactor ‚Äî the emitted AQL is unchanged ‚Äî plus direct unit tests for both helpers.
- New AQL array "question mark" helper `oihana\arango\db\functions\arrays\arrayContains()` (`array[? <quantifier> FILTER cond]`): the boolean array-contains operator, the direct/idiomatic form of `LENGTH(array[* FILTER cond]) > 0`, with optional `ANY` / `ALL` / `NONE` / `AT LEAST (n)` quantifiers (omitted = at least one). Provided as a primitive (with direct unit tests) ‚Äî notably to express quantified conditions over arrays of objects (e.g. `doc.reviews[? AT LEAST (3) FILTER CURRENT.rating >= @v]`); the existing `LENGTH(...) > 0` existentials are left unchanged.
- Opt-in live ArangoDB integration suite (`#[Group('integration')]`, run via `vendor/bin/phpunit -c phpunit-integration.xml`): builds a disposable database from `configs/config.toml`, seeds it, executes the generated AQL for real, then drops it ‚Äî and skips cleanly (never fails) when no instance is reachable. Covers the facet builders and the `alt` transformations end-to-end, complementing the server-free unit suite that only freezes the AQL string.

### Changed

- `Facet::FIELD` is now operator-enabled (`FilterComparator`): it accepts an `op` (in the definition via `Facet::OP`, or per request via an `{op, val}` object), defaulting to `match` (`=~`) for backward compatibility, with generic negation. Numeric scalars keep their type, so `doc.price >= 100` never binds a string.
- The list facet family is unified under `Facet::IN` (op-driven membership); `Facet::LIST`, `LIST_FIELD` and `LIST_FIELD_SORTED` become thin aliases, and the broken `length` branch was removed. The facet dispatcher was hardened (nullable type lookup, null-safe logger).

### Removed

- `PositionTrait` (`models/traits/aql/`) ‚Äî dead, half-written trait: it was composed by no model, `preparePosition()` was never called, and its only AQL form had a hardcoded `@value` placeholder (no `$binds`/value parameter) plus an unfiltered `FOR doc_coll IN <collection>` scan. Removed the trait and its characterization test rather than freeze unused, broken output; a position-ranking feature, if ever needed, should be designed fresh with proper binding.
- `Facet::THESAURUS` ‚Äî it was exactly an `EDGE` facet with `AQL::FIELDS = '_key,name,alternateName'` and `op = like`, so it is now expressed as that configuration rather than a dedicated type.
- Magic `id ‚Üí _key` rewriting in facets ‚Äî key/property aliases are now declared explicitly via `Facet::PROPERTY` everywhere, with no hidden remapping.

### Fixed

- `CollectionManagementTrait` (`ArangoDB` fa√ßade) called an **undefined** `toLegacyException()` in the `catch` blocks of `collectionRename()`, `getIndex()` and `getIndexes()` ‚Äî a leftover shim from the migration off the official ArangoDB-PHP fork. On any client error those three methods crashed with `Error: Call to undefined method` instead of surfacing the failure. The dead `try/catch` wrappers are removed so the `oihana\arango\clients\exceptions\ArangoException` propagates directly, and the redundant `ArangoException as ArangoClientException` import alias (both names resolved to the same class) is dropped. The boolean / log-and-swallow methods are unchanged.
- `PropertyController` PATCH in `raw` mode (`init[Arango::RAW] = true`) returned `null` instead of the submitted property value: the handler read the prepared payload with object-property syntax (`$payload->{$this->property}`), but `PayloadsTrait::propertyPayload()` yields an **array** (`[ property => value ]`), so the access always resolved to `null`. It now reads the value with `oihana\core\accessors\getKeyValue( (array) $payload , $this->property )`, which handles arrays and objects uniformly. Non-`raw` mode (which reloads via `model->get()`) is unchanged. Characterized then fixed in `PropertyControllerTest`.
- `aqlFields` `Field::QUOTED`: it double-quoted the key used for BOTH the output label and the attribute access, emitting invalid AQL like `"my-key":doc."my-key"` (an attribute with special characters cannot be reached with double quotes ‚Äî confirmed rejected by ArangoDB with a syntax error). The attribute access now uses backticks (`"my-key":doc.`my-key``), the valid form; only the output label is double-quoted. Cases pairing `Field::QUOTED` with `Field::NAME` or `Field::ALTERS` are unchanged. Characterized then fixed in `AqlFieldsTest`.
- AQL `like()` helper: its third parameter was named `$caseSensitive` but maps to ArangoDB's `LIKE(text, search, caseInsensitive)` third argument, so it did the opposite of its name (`true` produced a case-INSENSITIVE match). Renamed to `$caseInsensitive` (default `false` = case-sensitive, matching AQL). The emitted AQL is unchanged for every call path ‚Äî `?search` stays case-insensitive and the `like` filter-function alter keeps its output ‚Äî only the (now honest) parameter name and the docs change. Added the previously missing direct unit test, and fixed the stale `SearchTraitTest` comment that described the predicates as "case-sensitive".
- `command:arangodb` document `upsert` now honors the `removeKeys` exclusion list: it was passed under the `excludes` key, which the upsert query never reads, so the exclusion silently had no effect.
- `ArangoCommand` registered the wrong signals for event dispatch: it passed `(int) [SIGINT, ‚Ä¶]` (which casts to `1`) to the variadic `setSignalsToDispatchEvent()`. The intended signal list is now passed.
- Removed the unused `ignoreCollection` arangodump option (`ArangoDumpOptions::$ignoreCollection` / `ArangoDumpOption::IGNORE_COLLECTION`): no such flag exists in `arangodump`, and exclusion is resolved client-side as a `--collection` complement. The `--ignore-collection` CLI flag is unchanged.
- `ArangoConfigTrait` now validates the connection settings with exceptions instead of `assert()` (which is compiled out when `zend.assertions=-1`), so a missing database/endpoint/username never silently flows on as null. Properties are initialized; an empty password remains valid.
- Declare the previously-undeclared `ext-pcntl` requirement in `composer.json` ‚Äî `ArangoCommand`'s signal handling (`getSubscribedSignals()` / `handleSignal()` / `setSignalsToDispatchEvent()`) relies on the `SIGINT` / `SIGTERM` / `SIGUSR1` / `SIGUSR2` / `SIGALRM` constants provided by the extension.
- Security: `command:arangodb dump` / `restore` no longer run `arangodump` / `arangorestore` through a shell. The command was assembled as a single string (option values quoted only with `json_encode`) and passed to `system()`, so a value containing shell metacharacters (password, database, endpoint, directory, collection name) could break the command or inject arbitrary shell. The processes are now executed via `proc_open()` with an explicit argument vector (no shell), so values are passed verbatim. New `ArangoProcessTrait` (`optionsToArguments()` + `runProcess()`), covered by `ArangoProcessTraitTest` including a metacharacter-stays-one-argument unit test and a no-shell probe.
- `command:arangodb restore --last` could never locate the archive: it set the input file to the bare filename returned by `findFiles()` (whose `FILTER` yields `$file->getFilename()`), so `assertFile()` failed unless the current working directory happened to be the dump directory. The `--last` branch now prepends the dump directory, matching the interactive branch.

- `command:arangodb restore --date <date>` could never locate the archive. The by-date suffix was built as `Char::DASH . $database . $shouldEncrypt ? FileExtension::TAR_GZ_ENCRYPTED : FileExtension::TAR`, but `.` binds tighter than the ternary, so PHP evaluated `(Char::DASH . $database . $shouldEncrypt)` as the condition ‚Äî always a non-empty (truthy) string. The suffix therefore always collapsed to `.tar.gz.enc` and silently dropped the `-{database}` segment, so a non-encrypted restore-by-date never matched its `{date}-{database}.tar.gz` file. The non-encrypted branch also used `FileExtension::TAR` (`.tar`) instead of `FileExtension::TAR_GZ` (`.tar.gz`), which the dump action actually produces. The suffix logic is now isolated in the testable `ArangoRestoreAction::getArchiveFileSuffix()` helper and covered by `ArangoRestoreActionTest`.

- `Facet::ARRAY_COMPLEX` was broken at query time: it iterated an undefined variable (`FOR doc_workshops IN resultworkshops`, with no separator) and built its filter on the out-of-scope `result` document, so the facet never ran. It now iterates the in-scope embedded array (`doc.<key>`) and returns `1`. Confirmed against a live ArangoDB.
- `Facet::EDGE_COMPLEX` used a malformed loop variable (`AQL::DOC_REF . $key` ‚Üí e.g. `docRefnumbers`) instead of the sibling convention `doc_<key>`; it now reads `doc_<key>` like the other builders, and gained per-field multi-value (OR) plus per-value inline negation (`!=`) on the same traversed vertex.
- Date filters with a valid `tz` always threw: the timezone-conversion path passed the bound placeholder (`@tz`) to `dateLocalToUTC()`, whose own timezone re-validation then rejected it (the literal timezone is already validated upstream by `isValidTimezone()`). The re-validation is now disabled there (`$throwable = false`), so `?filter={...,"tz":"Europe/Paris"}` emits `DATE_LOCALTOUTC(@v,@tz)` instead of crashing ‚Äî and the new date `between` honors `tz` on both bounds.

### Security

- HTTP facets enforce a strict trust contract: only bound values (`@bind`) are user-controlled. Operators are whitelisted (`getAlias`, falling back to the type default), facet keys are whitelisted against the model's `Arango::FACETS` (an unknown key is ignored), and the URL-provided sub-field names of complex facets are validated by `assertAttributeName` before being interpolated ‚Äî a dangerous name fails the facet (logged and skipped). `alt` function names are likewise whitelisted (an unknown function is a no-op), so no `?filter=` / `?facets=` input can inject AQL.
