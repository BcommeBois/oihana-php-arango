# AQL options reference

The [`api/src/oihana/arango/db/options/`](../../../api/src/oihana/arango/db/options/) folder groups **15 classes** that model the options of every AQL operation and every index type. These are hydratable, validated and serializable objects — intermediate products between the application declaration (a PHP array) and the `OPTIONS { ... }` AQL clause generated by [`aqlOptions()`](aql/aql-operations.md#aqloptions).

## The pattern

All option classes share the same mechanic:

1. **Hydratable from an array**: `new SomeOptions([ 'useCache' => true ])` initializes properties from an associative *array*. Unknown keys are silently ignored (forward-compat with future ArangoDB options).
2. **JsonSerializable**: `json_encode($options)` produces the JSON `{"useCache":true,...}` consumed by the ArangoDB engine.
3. **`null` value filtering**: undefined properties are not emitted, which avoids overriding a server default with an explicit `null`.
4. **Wired to `aqlOptions()`**: the host operation (`aqlFor`, `aqlInsert`, ...) consumes the appropriate options class and delegates the conversion to `OPTIONS { ... }` to `aqlOptions()`.

In application use, you rarely pass an instance directly — an *array* with `AQL::*` keys is enough, the framework instantiates the matching `Options` internally.

```php
// Typical form (array hydrated inside aqlFor)
aqlFor
([
    AQL::DOC_REF => 'doc' ,
    AQL::IN      => 'users' ,
    AQL::OPTIONS =>
    [
        'indexHint'      => 'byEmail'  ,
        'forceIndexHint' => true       ,
        'useCache'       => true       ,
    ] ,
]) ;

// Equivalent with explicit instance
$opts = new ForOptions
([
    'indexHint'      => 'byEmail' ,
    'forceIndexHint' => true      ,
]) ;

aqlFor( [ AQL::DOC_REF => 'doc' , AQL::IN => 'users' , AQL::OPTIONS => $opts ] ) ;
```

## Options per operation

Nine classes each target a specific AQL operation.

| Class | Host operation | Main options (excerpt) |
|---|---|---|
| `QueryOptions` | All (at the query level) | `cache`, `count`, `fullCount`, `maxRuntime`, `memoryLimit`, `profile`, `stream` |
| `ForOptions` | `aqlFor` (loop) | `indexHint`, `forceIndexHint`, `disableIndex`, `useCache`, `lookahead` |
| `CollectOptions` | `aqlCollect` (aggregation) | `method` (`sorted`, `hash`) |
| `InsertOptions` | `aqlInsert` | `ignoreErrors`, `waitForSync`, `overwriteMode`, `keepNull`, `mergeObjects` |
| `UpdateOptions` | `aqlUpdate` | `ignoreErrors`, `keepNull`, `mergeObjects`, `waitForSync`, `ignoreRevs`, `exclusive` |
| `ReplaceOptions` | `aqlReplace` | `ignoreErrors`, `waitForSync`, `ignoreRevs`, `exclusive` |
| `RemoveOptions` | `aqlRemove` | `ignoreErrors`, `waitForSync`, `ignoreRevs`, `exclusive` |
| `UpsertOptions` | `aqlUpsert` | Combination of Insert + Update (`ignoreErrors`, `keepNull`, `mergeObjects`, `exclusive`, ...) |
| `TraversalOptions` | `aqlTraversal` | `order` (`bfs`, `dfs`, `weighted`), `uniqueVertices`, `uniqueEdges`, `bfs` (deprecated) |

> Each class exposes the official options of the corresponding ArangoDB operation. When ArangoDB adds an option (e.g. `parallelism` for traversals), it is enough to add the property in the class — no other layer to touch.

### Targeted examples

**Insertion with conflict handling:**

```php
use oihana\arango\db\enums\OverwriteMode ;

aqlInsert
([
    AQL::COLLECTION => 'users'                     ,
    AQL::DOCUMENT   => $document                   ,
    AQL::OPTIONS    =>
    [
        'ignoreErrors'   => true                   ,
        'overwriteMode'  => OverwriteMode::IGNORE  ,
        'waitForSync'    => false                  ,
    ] ,
]) ;
```

**Query with *full count* (for pagination):**

```php
$db->prepare
([
    'query'    => 'FOR doc IN users LIMIT @offset, @count RETURN doc' ,
    'bindVars' => [ 'offset' => 0 , 'count' => 50 ] ,
    'options'  => [ 'fullCount' => true ] ,
])
   ->execute() ;

$total = $db->getFoundRows() ; // total before LIMIT
```

**Breadth-first traversal, unique vertices:**

```php
use oihana\arango\db\enums\options\TraversalOrder ;
use oihana\arango\db\enums\options\TraversalUniqueVertices ;

aqlTraversal
([
    AQL::START     => '@startVertex' ,
    AQL::GRAPH     => 'social'        ,
    AQL::DIRECTION => Traversal::OUTBOUND ,
    AQL::OPTIONS   =>
    [
        'order'          => TraversalOrder::BFS         ,
        'uniqueVertices' => TraversalUniqueVertices::PATH ,
    ] ,
]) ;
```

## Index options — `options/indexes/`

Six classes model the index creation options, consumed by [`CollectionManagementTrait::createIndex()`](quickstart.md#manage-indexes).

| Class | Index type | Main options |
|---|---|---|
| `IndexOptions` | Abstract base | `fields`, `name`, `inBackground`, `sparse` |
| `PersistentIndexOptions` | `persistent` | `fields`, `unique`, `sparse`, `name`, `deduplicate`, `inBackground`, `cacheEnabled`, `estimates` |
| `TTLIndexOptions` | `ttl` | `fields`, `expireAfter`, `name`, `inBackground`, `selectivityEstimate` |
| `GeoIndexOptions` | `geo` | `fields`, `geoJson`, `name`, `inBackground`, `legacyPolygons` |
| `MDIIndexOptions` | `mdi` (multi-dimensional) | `fields`, `fieldValueTypes`, `unique`, `sparse`, `storedValues` |
| `VectorIndexOptions` | `vector` (ArangoDB 3.12+) | `fields`, `params` (with sub-`FaithParam` for Faiss), `cacheEnabled` |

### Examples

**Unique `persistent` index for emails:**

```php
use oihana\arango\db\options\indexes\PersistentIndexOptions ;

$index         = new PersistentIndexOptions() ;
$index->fields = [ 'email' ]    ;
$index->unique = true           ;
$index->sparse = true           ;

$db->createIndex( 'users' , $index ) ;
```

**TTL index for automatic session expiration:**

```php
use oihana\arango\db\options\indexes\TTLIndexOptions ;

$ttl              = new TTLIndexOptions() ;
$ttl->fields      = [ 'expiresAt' ] ;
$ttl->expireAfter = 0               ;        // expire as soon as expiresAt < now

$db->createIndex( 'sessions' , $ttl ) ;
```

**`geo` index on GeoJSON coordinates:**

```php
use oihana\arango\db\options\indexes\GeoIndexOptions ;

$geo          = new GeoIndexOptions() ;
$geo->fields  = [ 'location' ] ;
$geo->geoJson = true            ;

$db->createIndex( 'places' , $geo ) ;
```

**`vector` index for similarity search:**

```php
use oihana\arango\db\options\indexes\VectorIndexOptions ;
use oihana\arango\db\enums\FaithParam ;

$vec          = new VectorIndexOptions() ;
$vec->fields  = [ 'embedding' ] ;
$vec->params  =
[
    'dimension'    => 768          ,
    'metric'       => 'cosine'     ,
    FaithParam::NLISTS => 1000     ,
] ;

$db->createIndex( 'documents' , $vec ) ;
```

## Serialization and inspection

Every options class implements `JsonSerializable`. This allows using it with `json_encode()`, storing it in cache, or *logging* it for audit:

```php
$opts = new ForOptions( [ 'indexHint' => 'byEmail' , 'useCache' => true ] ) ;
echo json_encode( $opts ) ;
// {"indexHint":"byEmail","useCache":true}
```

The internal `aqlOptions()` does this `json_encode` then wraps the result in `OPTIONS { ... }`.
